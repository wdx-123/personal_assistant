12、OJTask 一步式无感解析已落地

**当前实现**
1、OJTask 已经切到“一步式无感解析”：
 - 用户侧输入统一为 `platform + title`
 - `analyze / create / update / revise` 共用同一套本地解析服务
 - 不再要求用户传 `ack_pending / type / question_id / question_code`
2、题库与任务状态已经拆开：
 - 平台题库表只保留权威题目
 - 任务项保存“输入意图 + 解析快照”
 - 待解析事实落在 `oj_question_intakes`
3、解析状态已经收敛为：
 - `resolved`
 - `pending_resolution`
 - `invalid`
4、题库同步成功后会发 `question_upserted` 事件：
 - 自动扫描并回填 `pending_resolution` 的任务项
 - 不再向题库写 shadow row
5、preFlight 仍然保留，但已经降级为最后兜底：
 - 执行前再做一次“本地唯一命中”尝试
 - 仍然失败则转 `invalid`
 - 执行结果按 `question_not_found` 短路

**当前请求口径**
1、`POST /oj/task/analyze`
 - 请求体改为 `items[]`
 - 每项只传：
   - `platform`
   - `title`
2、`POST /oj/task`
3、`PUT /oj/task/{id}`
4、`POST /oj/task/{id}/revise`
 - `items[]` 统一只传：
   - `platform`
   - `title`
   - `analysis_token?`
 - `analysis_token` 只在 Analyze 返回多候选，需要显式确认时才传

**当前响应口径**
1、Analyze 响应仍是三类：`resolved / ambiguous / missing`
 - `resolved[]` 每项返回：
   - `input_index`
   - `platform`
   - `title`
   - `analysis_token`
   - `resolved_question_id`
   - `resolved_question_code`
   - `resolved_title`
 - `ambiguous[]` 每项返回：
   - `input_index`
   - `platform`
   - `title`
   - `options[]`
 - `ambiguous.options[]` 每项返回：
   - `analysis_token`
   - `resolved_question_id`
   - `resolved_question_code`
   - `resolved_title`
 - `missing[]` 每项只返回：
   - `input_index`
   - `platform`
   - `title`
2、任务详情与任务项响应统一为：
 - 一定会返回：
   - `platform`
   - `input_title`
   - `input_mode`
   - `resolution_status`
   - `resolution_note`
 - 仅已解析题目返回：
   - `resolved_question_id`
   - `resolved_question_code`
   - `resolved_title_snapshot`
3、执行详情中的 `completed_items / pending_items` 也统一为同一套口径：
 - 一定会返回：
   - `platform`
   - `input_title`
   - `resolution_status`
   - `resolution_note`
   - `result_status`
   - `reason`
 - 仅已解析题目返回：
   - `resolved_question_id`
   - `resolved_question_code`
   - `resolved_title_snapshot`

**已废弃口径**
1、不再对外暴露或要求：
 - `ack_pending`
 - `type=new`
 - `question_id`
 - `question_code`
 - `platform_question_id`
 - `question_title_snapshot`
 - `question_source_status`
 - `shadow_pending`
 - `shadow_resolved`
2、UI 展示应改为直接看：
 - `resolution_status`
 - `resolution_note`
 - `resolved_*`
